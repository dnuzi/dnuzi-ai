## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/dnuzi/niyox.git
   cd niyox
   ```
3. **Install** dependencies:
   ```bash
   npm install
   ```
4. **Build** the library:
   ```bash
   npm run build
   ```

## How to Contribute

### Reporting Bugs

Before opening an issue, please:

- Search [existing issues](https://github.com/dnuzi/niyox/issues) to avoid duplicates.
- Reproduce the bug on the latest version.

When filing a bug report, include:

- **Node.js version**
- **Package version** (`niyox --version`)
- **Steps to reproduce** (minimal, complete example)
- **Expected behaviour** vs **actual behaviour**
- **Error output / stack trace** if available

Use the **Bug Report** issue template.

---

### Suggesting Features

Feature requests are welcome! Please:

- Check whether the feature has already been requested.
- Describe the problem the feature solves.
- Provide examples of the desired API / CLI interface if relevant.

Use the **Feature Request** issue template.

---

### Submitting a Pull Request

1. Make sure your branch is up to date with `main`:
   ```bash
   git fetch upstream
   git rebase upstream/main
   ```
2. Run the full test suite and confirm it passes:
   ```bash
   npm test
   ```
3. Add or update tests for any code you change.
4. Update documentation (README, JSDoc comments, etc.) as needed.
5. Push your branch and open a Pull Request against `main`.
6. Fill in the PR template — description, related issue(s), screenshots if relevant.

A maintainer will review your PR. Please be patient; all feedback is meant to be constructive.

## Development Setup

```
niyox/
├── bin/
│   └── cli.js              CLI entry point — REPL, /mongo <url>, one-shot
├── src/
│   ├── client.js           Core HTTP client + in-memory history
│   └── storage.js          MongoDB persistence (custom URI support)
├── lib/                    Built output (generated by npm run build)
├── react/
│   └── useNiyoX.js         useNiyoX hook, <NiyoXChat>, <NiyoXProvider>
├── examples/
│   ├── react-cra/          Create React App / Vite example
│   ├── nextjs/             Next.js App Router example
│   └── vite/               Vite vanilla JS example
├── html/
│   └── index.html          Browser demo (zero dependencies)
├── test/                   Jest test suites
└── scripts/
    └── build.js            Assembles lib/ from src/
```

**Useful commands:**

| Command | Description |
|---|---|
| `npm run build` | Build `lib/` from `src/` |
| `npm test` | Run all tests with coverage |
| `npx jest --watch` | Run tests in watch mode |
| `npx jest test/client.test.js` | Run a single test file |

## Project Structure

| Layer | File | Responsibility |
|---|---|---|
| CLI | `bin/cli.js` | Interactive REPL + one-shot + `/mongo <url>` |
| SDK (high-level) | `lib/index.cjs` | `NiyoXAI` — combines client + storage |
| HTTP client | `src/client.js` | Raw API calls, in-memory history |
| Persistence | `src/storage.js` | Optional MongoDB layer, custom URI support |
| React hook | `react/useNiyoX.js` | `useNiyoX`, `<NiyoXChat>`, `<NiyoXProvider>` |
| Browser | `html/index.html` | Self-contained browser demo |

## Coding Standards

**JavaScript / Node.js**
- Follow the existing code style: `"use strict"`, 2-space indent.
- **JSDoc** all public methods — include `@param` types and `@returns`.
- **No `console.log`** left in library code (`bin/cli.js` output is fine).
- **No new runtime dependencies** without prior discussion in an issue.
- Keep functions small and single-purpose.

**React (`react/useNiyoX.js`)**
- Must remain a single self-contained file — no external imports beyond React.
- All components must work without a bundler (plain `<script type="module">`).
- Include `"use client"` at the top for Next.js App Router compatibility.

## Testing

Tests live in `test/` and use **Jest**.

| File | Covers |
|---|---|
| `test/client.test.js` | `NiyoXClient` unit tests (fetch-mocked, no real network calls) |
| `test/sdk.test.js` | `NiyoXAI` + `NiyoXStorage` integration tests |
| `test/cli.test.js` | CLI binary smoke tests (`--version`, `--help`) |

**Adding tests:**

- Every new feature needs at least one happy-path and one error-path test.
- Tests must not make real network requests — mock `globalThis.fetch` as shown in existing tests.
- For new MongoDB methods in `NiyoXStorage`, add offline (disabled) tests to `sdk.test.js`.
- For custom `mongoUri` / `dbName` support, test that the correct URI is passed through.
- Aim to keep coverage above **90 % statements**.

```bash
# Run with coverage report
npx jest --coverage --forceExit
```

## Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>(<scope>): <short summary>

[optional body]

[optional footer]
```

Common types: `feat`, `fix`, `docs`, `test`, `refactor`, `chore`, `ci`.

**Scopes for 0.0.3+:** `storage`, `client`, `cli`, `react`, `examples`, `build`.

Examples:

```
feat(storage): support custom mongoUri and dbName options
feat(react): add useNiyoX hook and NiyoXChat component
fix(cli): persist custom mongoUri across sessions via conf
docs(readme): add React / Next.js / Vite usage examples
test(sdk): add coverage for custom MongoDB URI path
chore(build): bump version to 0.0.3
```
