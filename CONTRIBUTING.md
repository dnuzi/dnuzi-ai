## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/<your-username>/dnuzi-ai.git
   cd dnuzi-ai
   ```
3. **Install** dependencies:
   ```bash
   npm install
   ```
4. **Create a branch** for your work:
   ```bash
   git checkout -b feat/your-feature-name
   ```

## How to Contribute

### Reporting Bugs

Before opening an issue, please:

- Search [existing issues](https://github.com/dnuzi/dnuzi-ai/issues) to avoid duplicates.
- Reproduce the bug on the latest version.

When filing a bug report, include:

- **Node.js / Python version**
- **Package version** (`dnuzi --version`)
- **Steps to reproduce** (minimal, complete example)
- **Expected behaviour** vs **actual behaviour**
- **Error output / stack trace** if available

Use the **Bug Report** issue template.

---

### Suggesting Features

Feature requests are welcome! Please:

- Check whether the feature has already been requested.
- Describe the problem the feature solves.
- Provide examples of the desired API / CLI interface if relevant.

Use the **Feature Request** issue template.

---

### Submitting a Pull Request

1. Make sure your branch is up to date with `main`:
   ```bash
   git fetch upstream
   git rebase upstream/main
   ```
2. Run the full test suite and confirm it passes:
   ```bash
   npm test
   ```
3. Add or update tests for any code you change.
4. Update documentation (README, JSDoc comments, etc.) as needed.
5. Push your branch and open a Pull Request against `main`.
6. Fill in the PR template — description, related issue(s), screenshots if relevant.

A maintainer will review your PR. Please be patient; all feedback is meant to be constructive.

## Development Setup

```
dnuzi-ai/
├── bin/          CLI entry point  (cli.js)
├── src/          Source files
│   ├── client.js     Core HTTP client
│   ├── storage.js    MongoDB storage layer
│   └── index.js      High-level DnuziAI class + exports
├── lib/          Built output  (generated by npm run build)
├── test/         Jest test suites
├── python/       Python SDK  (dnuzi.py)
├── html/         Browser demo  (index.html)
└── scripts/      Build scripts
```

**Useful commands:**

| Command | Description |
|---|---|
| `npm test` | Run all tests with coverage |
| `npm run build` | Build `lib/` from `src/` |
| `npx jest --watch` | Run tests in watch mode |
| `npx jest test/client.test.js` | Run a single test file |

## Project Structure

| Layer | File | Responsibility |
|---|---|---|
| CLI | `bin/cli.js` | Interactive REPL + one-shot commands |
| SDK (high-level) | `src/index.js` | `DnuziAI` — combines client + storage |
| HTTP client | `src/client.js` | Raw API calls, in-memory history |
| Persistence | `src/storage.js` | Optional MongoDB layer |
| Python SDK | `python/dnuzi.py` | Mirror of the JS SDK in Python |
| Browser | `html/index.html` | Self-contained browser demo |

## Coding Standards

- **Style:** follow the existing code style (no semicolons where omitted, 2-space indent, `"use strict"`).
- **No new runtime dependencies** without prior discussion in an issue.
- **JSDoc** all public methods.
- **No `console.log`** left in library code (CLI output is fine).
- Keep functions small and single-purpose.
- Python code should follow PEP 8 and include type hints on public methods.

---

## Testing

Tests live in `test/` and use **Jest**.

- `test/client.test.js` — unit tests for `DnuziClient` (no real network calls; uses `globalThis.fetch` mock).
- `test/sdk.test.js` — integration tests for `DnuziAI` and `DnuziStorage`.
- `test/cli.test.js` — smoke tests for the CLI binary.

**Adding tests:**

- Every new feature needs at least one happy-path and one error-path test.
- Tests must not make real network requests — mock `globalThis.fetch` as shown in existing tests.
- Aim to keep coverage above **90 % statements**.

```bash
# Run with coverage report
npx jest --coverage --forceExit
```

## Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>(<scope>): <short summary>

[optional body]

[optional footer]
```

Common types: `feat`, `fix`, `docs`, `test`, `refactor`, `chore`, `ci`.

Examples:

```
feat(storage): add deleteConversation method
fix(client): handle timeout correctly on Node 18
docs(readme): update Python install instructions
test(sdk): add coverage for disabled-storage paths
```
